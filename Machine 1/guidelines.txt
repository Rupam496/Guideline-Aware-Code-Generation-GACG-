## 1. Indentation
Priority: LOW
Category: Formatting
AST: Layout Rule

Use 4 spaces per indentation level.

Use aligned continuation lines or hanging indents.

### Correct:
foo = long_function_name(
    var_one,
    var_two
)

### Wrong:
foo = long_function_name(
  var_one,
  var_two
)

---

## 2. Line Length
Priority: LOW
Category: Formatting
AST: Layout Rule

Prefer lines under 79 characters when practical.

### Correct:
x = 10

### Wrong:
x = "This is a very long line that unnecessarily exceeds recommended width for readability"

---

## 3. Naming Conventions
Priority: MEDIUM
Category: Readability
AST: FunctionDef/ClassDef/Assign(name style)

Functions and variables should use snake_case.
Classes should use PascalCase.
Constants should use UPPER_CASE.

### Correct:
def my_function():
    pass

class MyClass:
    pass

MAX_VALUE = 10

### Wrong:
def MyFunction():
    pass

class my_class:
    pass

maxValue = 10

---

## 4. Imports
Priority: MEDIUM
Category: Maintainability
AST: Import / ImportFrom

Use separate import lines and group:
1. Standard library
2. Third-party
3. Local modules

### Correct:
import os
import sys

import numpy as np

from mypkg import helper

### Wrong:
import sys, os
import numpy, mypkg

## 8. Nesting Depth
Priority: MEDIUM
Category: Maintainability
AST: Nested If/For/While depth

Avoid deeply nested code. Prefer max depth around 3.

### Correct:

## 14. Docstrings

Priority: MEDIUM
Category: Documentation
AST: FunctionDef/ClassDef without docstring

Public or non-trivial modules, functions, classes, and methods should include concise docstrings that explain purpose, important arguments, return values, side effects, or usage when helpful.

Use one-line docstrings for simple cases and multi-line docstrings for more complex behavior.

Avoid redundant docstrings that merely repeat the function name.

### Correct:

def compute_area(radius: float) -> float:
    """Return the area of a circle for the given radius."""
    return 3.14159 * radius ** 2


def fetch_data(url: str, timeout: int) -> dict:
    """
    Fetch JSON data from the given URL.

    Args:
        url: Target URL.
        timeout: Request timeout in seconds.

    Returns:
        Parsed JSON response.
    """
    pass


class UserService:
    """Provide user-related business operations."""

### Wrong:

def compute_area(radius):
    return 3.14159 * radius ** 2


def fetch_data(url, timeout):
    """fetch data"""
    pass


def save_file(path, content):
    """save_file"""
    pass


class UserService:
    pass

## 16. Return Statements
Priority: HIGH
Category: Logic Bug
AST: Mixed Return(value / bare return)

Use consistent return behavior.

### Correct:

        return x
    return 0

### Wrong:
def f(x):
    if x > 0:
        return x
    return

## 19. Exception Handling
Priority: HIGH
Category: Error Handling
AST: ExceptHandler(type=None)

Catch specific exceptions. Avoid bare except.

### Correct:
try:
    x = int(s)
except ValueError:
    pass

### Wrong:
try:
    x = int(s)
except:
    pass

---

## 20. Exception Chaining
Priority: HIGH
Category: Error Handling
AST: Raise without cause inside except

Use `raise ... from e`.

### Correct:
try:
    work()
except IOError as e:
    raise RuntimeError() from e

### Wrong:
try:
    work()
except IOError:
    raise RuntimeError()

---

## 21. Context Managers
Priority: HIGH
Category: Resource Safety
AST: open() without with

Use context managers for files/resources.

### Correct:
with open("a.txt") as f:
    data = f.read()

### Wrong:
f = open("a.txt")
data = f.read()
f.close()

---

## 22. List Comprehensions
Priority: MEDIUM
Category: Pythonic Style
AST: map/filter(lambda)

Prefer simple comprehensions over map/filter with lambda.

### Correct:

### Wrong:
vals = list(map(lambda x: x*x, nums))

## 29. Mutable Default Arguments
Priority: HIGH
Category: Runtime Bug
AST: FunctionDef(default=List|Dict|Set)

Use None then create mutable object inside function.

### Correct:
def add(x, items=None):
    if items is None:
        items = []
    items.append(x)

### Wrong:
def add(x, items=[]):
    items.append(x)

### Correct:

## 31. Section Comments

Priority: MEDIUM
Category: Readability
AST: FunctionDef(long body without comments)

Functions longer than 10 lines should use short section comments to separate logical steps.

### Correct:
def process_orders(orders):
    # Validate input
    if not orders:
        return []

    # Transform records
    cleaned = [o.strip() for o in orders]

    # Save results
    return cleaned

### Wrong:
def process_orders(orders):
    if not orders:
        return []
    cleaned = [o.strip() for o in orders]
    return cleaned

---

## 32. Early Returns

Priority: HIGH
Category: Maintainability
AST: Deep nested If blocks

Prefer early returns to reduce nesting and improve readability.

### Correct:
def divide(a, b):
    if b == 0:
        return None
    return a / b

### Wrong:
def divide(a, b):
    if b != 0:
        return a / b
    else:
        return None

---

## 34. Main Guard

Priority: MEDIUM
Category: Python Best Practice
AST: Top-level executable script without main guard

Executable scripts should use main() and if __name__ == "__main__".

### Correct:
def main():
    run_program()

if __name__ == "__main__":
    main()

### Wrong:
run_program()

---

## 35. Explicit File Encoding

Priority: MEDIUM
Category: Reliability
AST: open() without encoding

Specify encoding="utf-8" for text files unless another encoding is required.

### Correct:
with open("data.txt", "r", encoding="utf-8") as file:
    content = file.read()

### Wrong:
with open("data.txt", "r") as file:
    content = file.read()

---

## 36. Logging Over Print

Priority: MEDIUM
Category: Production Readiness
AST: print() in utility/service code

Prefer logging instead of print statements outside tiny scripts.

### Correct:
import logging

logger = logging.getLogger(__name__)
logger.info("Processing started")

### Wrong:
print("Processing started")
